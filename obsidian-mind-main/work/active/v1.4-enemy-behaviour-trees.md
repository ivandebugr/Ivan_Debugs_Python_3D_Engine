# v1.4 ‚Äî Enemy Behaviour Trees

Blocked on: [[work/active/v1.3-asset-import-pipeline]]. See [[work/archive/version-map]] for sequencing.

## Goal

Replace the single-behaviour enemy (detect ‚Üí shoot) with a composable behaviour tree system. Each enemy owns a top-down evaluated tree that is ticked once per frame. Built-in leaf nodes cover patrol, chase, attack, flee, and idle. Composition is handled by standard combinators. Trees are configured per-type in `level.json` and `mod.json` ‚Äî no Python subclassing needed for common enemy variants. This feature was unblocked by the v1.1 fix that wired `CollisionManager.query_layer()` (see [[work/audits/2026-05-20-full-audit]]).

## Architecture ‚Äî Three Layers

### Layer 1 ‚Äî `Scripts/behaviour_tree.py` node primitives

```python
class Status(Enum):
    SUCCESS = "success"
    FAILURE = "failure"
    RUNNING  = "running"

class BehaviourNode:
    def tick(self, enemy, dt) -> Status: ...

# Compositors
class Sequence(BehaviourNode):
    """Tick children left-to-right; return FAILURE on first child FAILURE, else SUCCESS when all succeed."""

class Selector(BehaviourNode):
    """Tick children left-to-right; return SUCCESS on first child SUCCESS, else FAILURE when all fail."""

class Parallel(BehaviourNode):
    """Tick all children every frame; return SUCCESS when all succeed."""

# Decorators
class Invert(BehaviourNode):
    """Inverts child status: SUCCESS‚ÜíFAILURE, FAILURE‚ÜíSUCCESS, RUNNING‚ÜíRUNNING."""

class Repeat(BehaviourNode):
    """Tick child n times (n=-1 for infinite). Returns RUNNING while repeating."""

class Cooldown(BehaviourNode):
    """Prevents child from ticking more than once per `seconds` interval."""
```

`BehaviourNode` must not import from `main.py`, `enemy.py`, or any Ursina module. It is a pure data-structure layer ‚Äî all game state access happens through the `enemy` argument passed into `tick()`.

### Layer 2 ‚Äî Built-in leaf nodes

Each node's `tick(enemy, dt)` logic:

**`IdleNode`** ‚Äî always returns `SUCCESS`. Used as the final fallback in a `Selector` to ensure the tree never leaves a RUNNING state indefinitely. Zero CPU cost.

**`PatrolNode(waypoints: list[Vec3])`** ‚Äî moves the enemy toward `waypoints[current_index]`. Uses the same wall-avoidance pattern as `_swept_blocked` in `player_controller.py` ‚Äî either via a shared utility extracted from the player controller, or by calling `raycast()` directly (do NOT duplicate the swept logic). Returns `RUNNING` while moving toward a waypoint. Returns `SUCCESS` when the waypoint is reached, then advances `current_index`. Loops back to index 0 after the last waypoint.

**`ChaseNode(detection_range: float)`** ‚Äî checks player distance via `(enemy.position - player.position).length()`. If within `detection_range`: moves enemy toward player using the same stepped movement as the player controller; returns `RUNNING`. If out of range: returns `FAILURE` (causes parent `Sequence` to abort). Does not shoot ‚Äî that is `AttackNode`'s responsibility.

**`AttackNode(attack_range: float, cooldown: float)`** ‚Äî checks distance to player. If within `attack_range` and cooldown has elapsed: calls `enemy.shoot()`, resets cooldown timer, returns `SUCCESS`. If within range but on cooldown: returns `RUNNING`. If out of range: returns `FAILURE`.

**`FleeNode(flee_threshold_hp: int, flee_range: float)`** ‚Äî active when `enemy.health < flee_threshold_hp`. Moves enemy away from player (opposite direction of chase vector) up to `flee_range` units. Returns `RUNNING` while fleeing. Returns `SUCCESS` when the enemy has opened enough distance. Returns `FAILURE` immediately if `enemy.health >= flee_threshold_hp` ‚Äî tree falls through to the next node.

### Layer 3 ‚Äî Tree configuration

**Default tree** (replaces current `Enemy.update()` detect/shoot block):
```python
Selector([
    Sequence([ChaseNode(100), AttackNode(30, 1.0)]),
    IdleNode()
])
```
This matches the current single-behaviour enemy exactly ‚Äî chase if in range, attack if close enough, idle otherwise.

**Named presets** via `BehaviourTreeFactory.build(preset_name, config) -> BehaviourNode`:

| Preset | Tree structure |
|---|---|
| `"default"` | `Selector([Sequence([ChaseNode(100), AttackNode(30, 1.0)]), IdleNode()])` |
| `"patrol_then_attack"` | `Selector([Sequence([ChaseNode(60), AttackNode(25, 0.8)]), PatrolNode(waypoints), IdleNode()])` |
| `"flee_when_low"` | `Selector([Sequence([FleeNode(30, 15), IdleNode()]), Sequence([ChaseNode(100), AttackNode(30, 1.0)]), IdleNode()])` |
| `"aggressive"` | `Selector([Sequence([ChaseNode(150), AttackNode(40, 0.4)]), IdleNode()])` |

**`level.json` optional `"behaviour"` field on enemy entries:**
```json
{
  "type": "enemy",
  "position": [5.0, 2.0, 1.0],
  "behaviour": {
    "tree": "patrol_then_attack",
    "waypoints": [[3, 1, 0], [7, 1, 0], [7, 1, -4]]
  }
}
```
Unknown preset name ‚Üí log warning, fall back to `"default"`. Absent `"behaviour"` key ‚Üí default tree. Fully backwards compatible with all existing `level.json` files.

`mod.json` enemy types include a `"behaviour"` field using the same preset-name syntax (see [[work/active/v2.0-release]]).

## Enemy Integration

Changes to `enemy.py`:

```python
class Enemy(AliveEntity):
    def __init__(self, behaviour_tree=None, **kwargs):
        super().__init__(**kwargs)
        register(self, Layers.ENEMY)
        self._tree = behaviour_tree or BehaviourTreeFactory.build("default", {})

    def update(self):
        if not self.alive:  # guard stays first ‚Äî this is non-negotiable
            return
        self._tree.tick(self, time.dt)  # replaces detect/shoot block
```

`Enemy.shoot()` and `_is_occluded()` are unchanged ‚Äî `AttackNode` and `ChaseNode` call them via the `enemy` argument. The `if not self.alive: return` guard remains the first line of `update()` ‚Äî no exceptions.

## Level JSON Schema

Full example with all optional v1.4 fields:
```json
{
  "type": "enemy",
  "position": [5.0, 2.0, 1.0],
  "enemy_type": "default",
  "hp": 100,
  "rotation_y": 0,
  "behaviour": {
    "tree": "patrol_then_attack",
    "waypoints": [[3, 1, 0], [7, 1, 0], [7, 1, -4]]
  }
}
```

All fields optional. Missing `"behaviour"` ‚Üí default tree. Missing `"hp"` ‚Üí class default (100). Missing `"rotation_y"` ‚Üí 0. Existing level files with no behaviour field continue to work identically.

## Performance + Constraints

- **O(depth) per enemy per frame** ‚Äî typical tree depth is 3‚Äì5 nodes; cost scales with enemy count, not tree complexity
- **`PatrolNode` movement** uses the same wall-avoidance approach as `player_controller.py:_swept_blocked` ‚Äî extract to a shared utility or import directly rather than duplicating raycast logic (see [[brain/Patterns]] for the no-duplication rule)
- **`BehaviourNode` must not import from `main.py`** ‚Äî circular import risk; access game state only through the `enemy` argument
- **No new collision authority** ‚Äî `PatrolNode` uses raw `raycast().hit` for wall detection, same as `_swept_blocked`
- **No name checks** ‚Äî CLAUDE.md Rule 1 applies inside node implementations; use `can_hit()` or `isinstance()` only where type identity is genuinely needed

## Implementation Order

Each step produces a testable checkpoint. Steps 1‚Äì2 preserve current enemy behaviour exactly.

1. `BehaviourNode` + `Status` enum + `Sequence` / `Selector` / `Parallel` compositors
2. `IdleNode` + `AttackNode` ‚Äî swap into `Enemy.update()` immediately; behaviour is identical to today
3. `ChaseNode` ‚Äî adds detection range cutoff
4. `PatrolNode` ‚Äî requires waypoint routing; test with a simple two-point path
5. `FleeNode` ‚Äî requires HP threshold check
6. Decorators: `Invert`, `Repeat`, `Cooldown`
7. `BehaviourTreeFactory` + named presets
8. `level.json` `"behaviour"` field support in `load_level()` (main.py) and `load_existing_level()` (level_editor.py)
9. Level editor inspector behaviour dropdown (preset selector + waypoint list for `patrol_then_attack`)
