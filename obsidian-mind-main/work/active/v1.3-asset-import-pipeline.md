# v1.3 — Asset Import Pipeline

Blocked on: [[work/active/v1.2-level-editor-overhaul]]. See [[work/archive/version-map]] for sequencing.

## Goal

Drop a `.png`, `.obj`, `.gltf`, or `.wav` into the project folder or drag it into the editor window and use it immediately — no manual config file editing. v1.3 also delivers asset hot-reload: texture changes on disk reflect in the editor without restarting. Stays entirely within Panda3D's native loader surface.

## Supported Formats

| Category | Formats | Notes |
|---|---|---|
| Textures | `.png`, `.jpg`, `.jpeg` | Panda3D native; discovery layer wraps `loader.loadTexture()` |
| Models | `.obj`, `.gltf`, `.glb`, `.egg` | `.obj` and `.egg` are Panda3D native; `.gltf`/`.glb` via `panda3d-gltf` (already in requirements) |
| Models | `.fbx` | **Not v1.3** — deferred to post-v2.0 parking lot; requires external converter tooling. See [[work/archive/version-map#Post-v2.0 Parking Lot]] |
| Audio | `.wav`, `.ogg` | `loader.loadSfx()` |

## Features

### 1. `AssetRegistry` (`Scripts/asset_registry.py`)

Pure registry class — no import from `main.py` or `level_editor.py`. Scans `assets/textures/`, `assets/models/`, `assets/sounds/` at startup and builds a `{name: path}` manifest per category. Polls `os.stat()` on all tracked files every 2 seconds; fires `on_texture_changed(name, path)` and `on_model_changed(name, path)` callbacks on mtime change. Persists the manifest to `assets/manifest.json` so subsequent startups avoid a full scan. Module-level singleton: `asset_registry = AssetRegistry()`.

`AssetRegistry` never imports Ursina — it is a pure Python I/O layer. The editor imports it and drives the callbacks.

### 2. Asset browser panel

Positioned at bottom 15% of the editor window, `parent=camera.ui`. Three tabs: **Textures | Models | Sounds**. Each tab renders a scrollable grid of 64×64 thumbnail cards showing the asset name. Click selects the asset. Double-click applies it to all currently selected entities (texture → `Entity.texture`, model → `Entity.model`). Model thumbnails are a placeholder cube icon in v1.3 — live 3D thumbnails are deferred. Browser panel is hidden when play-in-editor (F5) is active.

### 3. Asset hot-reload

When `AssetRegistry` fires `on_texture_changed(name, path)`:
1. `loader.loadTexture(path)` to force a fresh GPU upload (Panda3D caches by path; name-based lookup bypasses the stale cached handle)
2. Iterate all editor entities that reference this texture name; apply the new handle
3. Log a 1-line notice in the editor status bar

Model changes are flagged dirty and reloaded on next placement rather than live-replaced (replacing geometry mid-session risks state inconsistency). Hot-reload is disabled while play-in-editor mode is active — the game simulation owns those assets.

### 4. Drag-and-drop import

`base.win.setAcceptDrop(True)` + listener on the `drop-files` Panda3D event. On drop:
1. Detect file extension — route to `assets/textures/`, `assets/models/`, or `assets/sounds/`
2. Copy file to the correct subfolder (do not move — preserve the source)
3. Call `asset_registry.rebuild()` to update manifest and reload thumbnails
4. Display a 2-second success notification toast in the editor HUD
5. Unsupported extension: display a red error text notification (no crash, no modal)

### 5. Texture picker in inspector

The texture thumbnail in the v1.2 inspector panel (see [[work/active/v1.2-level-editor-overhaul#Track D — Unity-Feel Editor Features]]) becomes clickable. Clicking opens a floating overlay grid of all registered textures from `AssetRegistry`. Clicking a texture in the overlay applies it to all selected entities and pushes a `ChangeTextureCommand` onto the undo stack. Closes on click-outside or Escape. Replaces the hardcoded `toggle_texture()` cycle from v1.0.

### 6. Model picker in inspector

Same floating overlay pattern as the texture picker — triggered by clicking the model field in the inspector. Available for block entities only in v1.3; enemy model swapping is deferred to v1.4 (see [[work/active/v1.4-enemy-behaviour-trees]]). Pushes `ChangeModelCommand` onto the undo stack.

### 7. `level.json` model field

Optional `"model"` field on block entries, default `'cube'`. Fully backwards-compatible — existing `level.json` files without the field load as cube blocks as before. Both `load_level()` in `main.py` and `load_existing_level()` in `level_editor.py` updated to read and apply the field.

```json
{
  "type": "block",
  "position": [0, 0, 0],
  "model": "assets/models/wall_pillar.obj"
}
```

Paths in `level.json` are always relative to the project root — never absolute. The loader resolves `assets/models/...` via `AssetRegistry`'s path map.

## Final Integration Audit — 2026-06-26

Cross-step integration check before marking shipped (re-derived from code, not session logs). All 7 steps **PASS**; all PART-2 integration scenarios pass (texture→model apply in sequence with texture persisting; independent two-step undo; save/reload round-trip; old-format load with no warnings; picker-open vs drag mutual exclusion via the `asset_picker_open` input guard). Cleanup clean (no diagnostic prints, `assets/manifest.json` gitignored + untracked, no debug code).

> [!warning] Known risk carried forward (flagged, not blocking — ships per user decision)
> **Bare-string texture on the load path.** `entry['texture']` from `level.json` is assigned directly to `Entity.texture` at four sites (`main.py:168`, and `Scripts/level_editor.py` `_restore_editor_level`/`_spawn_gameplay_from_snapshot`/`load_existing_level`) — the same *shape* as the [[brain/Gotchas]] Step-4 model bug. It works today because Ursina's `load_texture` recursively globs the asset folders and resolves a bare filename (verified headless: `floor_stone.png` loads the real 64×64 image; `''` → no texture, no crash). It is **not** routed through `_resolve_texture` the way every model load site is routed through `_resolve_model`. Latent fragility: relies on texture filenames staying globally unique across `assets/` subfolders. Safe fix when convenient — mirror the model treatment at those four sites. See [[brain/Gotchas]].
>
> **Minor:** `main.py:588` sets `window.on_resize` — dead code (Ursina never calls it; see CLAUDE.md Rule 16). Harmless; the editor already uses the correct `update_aspect_ratio` wrap.

## Dependencies / Constraints

- Requires v1.2 inspector panel (floating overlays attach to inspector widget)
- `panda3d-gltf` already in requirements — no new dependency for `.gltf` support
- `AssetRegistry` never imports from `main.py` or any Ursina module — pure I/O layer
- Paths in `level.json` always relative — no machine-specific absolute paths committed
- Asset browser panel cleaned up (hidden, not destroyed) on play-in-editor entry; restored on exit
- `manifest.json` excluded from git (add to `.gitignore`) — regenerated on each run from actual `assets/` folder state
