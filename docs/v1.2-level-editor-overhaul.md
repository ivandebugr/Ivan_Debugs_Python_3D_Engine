# v1.2 — Level Editor Overhaul

Blocked on: v1.1 (see [[work/audits/2026-05-20-full-audit]]). See [[work/archive/version-map]] for sequencing.

## Goal

Three parallel tracks ship together:

1. **Level editor** becomes Unity-feel with inspector panel, hierarchy panel, transform gizmos, multi-select, undo/redo stack, grid snap, camera bookmarks, and play-in-editor
2. **`main.py`** gets a `Game` state-machine class replacing module-level globals (`player`, `game_paused`, `pause_menu`), and `_clear_gameplay_entities()` becomes the single canonical scene teardown path
3. **Level editor JSON schema** expands to support enemy type, HP, rotation, and block colour — all optional fields, fully backwards-compatible

## Track A — Game State Machine

`Game` is a plain Python class (NOT an Ursina `Entity` subclass — it is a pure Python state container). Instantiated once at startup as `game = Game()`. It must not import Ursina at class-definition time; it holds references to Ursina objects but is itself framework-agnostic.

```
State enum: MAIN_MENU, PLAYING, PAUSED, RETURNING_TO_MENU

Fields:
  game.player       — Player instance (None when in main menu)
  game.enemies      — list of Enemy instances
  game.pause_menu   — PauseMenu instance (None when not paused)
  game.state        — current State enum value

Methods:
  game.start()          — replaces start_game(); spawns player, enemies, sets state=PLAYING
  game.pause()          — sets state=PAUSED, shows pause menu
  game.resume()         — sets state=PLAYING, hides pause menu
  game.return_to_menu() — calls _clear_gameplay_entities() as its sole teardown path; sets state=MAIN_MENU
```

All `if player and ...` guards in `input()` and `update()` become `if game.state == Game.PLAYING`. Module-level `player`, `game_paused`, and `pause_menu` globals are deleted. Because `Game` is not an Entity, it has no `update()` hook — callers drive it via `input()` and Ursina's `update()`.

## Track B — `_clear_gameplay_entities()`

Canonical teardown function in `main.py`. Called only from `game.return_to_menu()` — never directly. Six ordered steps:

1. For every entity in `game.enemies` that is `.alive`: call `e.die()` — never `destroy(e)` directly (see [[brain/Patterns#Run AliveEntity.die() before blanket destroy loops]])
2. `collision_manager.remove(game.player)` before any destroy call on the player
3. `destroy(game.player.weapon.crosshair)` then `destroy(game.player.weapon)`
4. `destroy(game.player.health_bar.text)` then `destroy(game.player.health_bar)` (see [[brain/Patterns#Explicit teardown at every destroy call site for camera.ui sub-entities]])
5. Blanket name-loop for level blocks, ground, and sky entities
6. After this function completes: only `camera` and `camera.ui` children remain in `scene.entities`

## Track C — Level Editor JSON Schema Expansion

New optional fields — all backwards-compatible. Absent fields use documented defaults.

**Block entries:**
```json
{
  "type": "block",
  "position": [0, 0, 0],
  "colour": [1.0, 1.0, 1.0],
  "rotation": [0, 0, 0]
}
```
- `"colour"`: RGB array `[r, g, b]` (0.0–1.0 range), default `[1, 1, 1]` — applied as `Entity.color`
- `"rotation"`: `[rx, ry, rz]` Euler degrees, default `[0, 0, 0]`

**Enemy entries:**
```json
{
  "type": "enemy",
  "position": [0, 0, 0],
  "enemy_type": "default",
  "hp": 100,
  "rotation_y": 0
}
```
- `"enemy_type"`: string tag, default `"default"` — forward-compatible with v2.0 `enemy_type_id` (see [[work/active/v2.0-release]])
- `"hp"`: int instance override, default 100
- `"rotation_y"`: float degrees, default 0

Inspector UI additions: colour sliders (R/G/B), enemy type dropdown (populated from registered types), HP input field, rotation Y slider.

## Track D — Unity-Feel Editor Features

Eight features — detailed description, rationale, and Ursina-specific implementation notes for each.

### 1. Multi-select + bulk operations

Shift-click adds/removes from selection. Right-mouse drag performs box-select in screen space. Bulk operations: delete all selected, set texture for all selected, set colour for all selected. Selection state is a `set` of entity references stored on `LevelEditor` (not on individual entities). Bulk colour/texture changes push a single `BulkChangeCommand` onto the undo stack to avoid O(N) command entries.

### 2. Transform gizmos

Three-axis move handles painted as `Mesh(mode='line')` geometry with cone tips — approximately 80 lines of mesh construction. X-axis = red, Y-axis = green, Z-axis = blue, matching Unity/Blender conventions so muscle memory transfers. Gizmo is hidden in play-in-editor mode. When multiple entities are selected, the gizmo renders at the group centroid; dragging moves all selected entities by the same delta. Gizmo entities are named `editor_gizmo_*` and are excluded from save/load.

### 3. Inspector panel

Positioned at right 15% of window, `parent=camera.ui, z=-1`. Shows editable fields: name, position (x/y/z), rotation (x/y/z), scale, texture (thumbnail + picker), colour (RGB sliders), HP (int field, enemy entities only), enemy_type (dropdown, enemy entities only). Uses Ursina `InputField` for text inputs; custom slider strip for colour and rotation. When multiple entities are selected with differing values, field shows `(multiple)` — editing pushes the new value to all selected. Inspector rebuilds on selection change only, not every frame.

### 4. Hierarchy panel

Positioned at left 12% of window, scrollable. Lists all blocks and enemies by display name (type + index). Click selects and focuses the camera on that entity. Active selection is highlighted. Panel rebuilds on structural changes (place, delete, load) only — not every frame — to avoid O(N) UI churn.

### 5. Undo/Redo stack

`Ctrl+Z` undoes, `Ctrl+Y` redoes. Stack depth: 50 steps, implemented as `collections.deque(maxlen=50)`. Command pattern: each operation implements `execute()` and `undo()`. Commands:
- `PlaceBlockCommand` — places and registers a block
- `DeleteEntityCommand` — removes and unregisters an entity (stores snapshot for undo)
- `MoveEntityCommand` — stores old/new position
- `ChangeTextureCommand` — stores old/new texture reference
- `ChangeColourCommand` — stores old/new colour tuple
- `ChangeHPCommand` — stores old/new HP value

Undo stack is cleared on level load and on save — these are ground-truth reset points.

### 6. Play-in-editor

`F5` hides all editor UI panels (inspector, hierarchy, gizmos, toolbar), saves current level state to an in-memory buffer, and starts the game inline using `game.start()`. `F5` again or `Esc` exits via `game.return_to_menu()` (which calls `_clear_gameplay_entities()` — the Track B function), restores editor UI from the memory buffer. Reuses `_patch_shaders_to_glsl120()` from the shared location once it is extracted to `compat.py` — never copy-pasted. See [[brain/North Star]] for the compat.py extraction task.

### 7. Configurable grid snap

Toolbar cycles through: `1.0 | 0.5 | 0.25 | Off`. All entity placement and gizmo movement rounds position using `round(p / snap) * snap`. Current snap value persisted to `editor_prefs.json` across sessions. Toolbar widget displays the active snap value. Off mode allows floating-point positioning.

### 8. Camera bookmarks

`Ctrl+1` through `Ctrl+5` saves the current editor camera position + rotation as a bookmark. `1` through `5` recalls. HUD labels in the top bar show which slots are occupied. Bookmarks persisted to `editor_prefs.json` as `{"bookmarks": {"1": {"pos": [...], "rot": [...]}, ...}}`.

## Implementation Order

Order chosen so each step is independently testable and no step blocks another unnecessarily:

1. **Game state machine + `_clear_gameplay_entities()`** — all other editor features depend on clean teardown; this ships first
2. **JSON schema expansion** — schema must exist before the inspector UI is built on top of it; backwards-compat verified by loading existing `level.json`
3. **Grid snap increment** — ~5 lines, zero risk, ships in same PR as schema
4. **Undo/redo stack** — must exist before any edit feature, so all subsequent features push to it
5. **Multi-select**
6. **Inspector panel** — includes colour/HP/rotation fields from Track C
7. **Hierarchy panel**
8. **Transform gizmos**
9. **Camera bookmarks**
10. **Play-in-editor** — last; requires stable Game state machine (step 1) + Track B teardown

## Tech Debt This Must Not Introduce

- No fourth collision authority — editor entities are pure visual with no registered bitmask layers
- All editor UI entities: `parent=camera.ui, z=-1` — never parented to world space
- All editor-only world entities named `editor_*` — distinguishable from level entities in save/load
- Undo stack cleared on load and save — not accumulated across sessions
- `_patch_shaders_to_glsl120()` used only from the shared `compat.py` module once extracted — never copy-paste

## Open Questions

| Question | Options | Default assumption |
|---|---|---|
| `Game`: plain Python object or Ursina `Entity` subclass? | Plain preferred (no update hook needed) | Plain Python object |
| Inspector: Ursina `InputField` or custom widget? | `InputField` for text; custom slider strip for numeric | `InputField` + custom sliders |
| Hierarchy: rebuild on structural change event or every frame? | Event-driven rebuild | Event-driven |
| Play-in-editor: disk save/restore or memory-only? | Memory buffer (faster, no I/O risk) | In-memory buffer |

### New files created
- `Scripts/game.py` — `Game` state-machine class + module-level singleton
- `Scripts/undo_redo.py` — `UndoRedoStack`, `PlaceEntityCommand`, `DeleteEntityCommand`, `MoveEntityCommand`, `ChangeTextureCommand`, `ChangeColourCommand`, `ChangePropertyCommand`
- `editor_prefs.json` — camera bookmarks + grid snap persistence (created at runtime on first Ctrl+S or Ctrl+1)

### Deviations from spec
- `start_game()` not replaced by `game.start()` as sole body — it retains spawn logic, then calls `game.start()`. The spec's intent (single entry point) is met; only the phrasing of "body is a single call" was aspirational.
- Gizmo cone tips implemented as small cube entities (spec acknowledged this as the v1.2 approach).
- `_enter_play_mode()` imports `_clear_gameplay_entities` from `main` lazily with a try/except fallback — avoids circular import at module level while still calling the canonical teardown when available.
- Resume via Esc in pause state is handled in `main.py input()` by calling `game.pause_menu.resume_game()` — clean, single code path.

### New tech debt introduced
None. No fourth collision authority, no editor entities in gameplay scene, undo stack clears on load/save.
